1. Verificar e instalar Python

Para poder programar en Python, es necesario tener instalado este lenguaje en nuestro equipo. En esta sección aprenderás a verificar si Python ya está instalado y, si no lo está, cómo instalarlo.

Verificar si Python está instalado

  1. Abre una terminal o consola de comandos:
    • En Windows, busca “cmd” en el menú de inicio.
    • En macOS o Linux, abre la aplicación “Terminal”.
  2. Escribe el siguiente comando y presiona Enter: 
    c:\> python --version   /  # python3 --version
  3. Si Python está instalado, verás un mensaje con la versión, por ejemplo: 
    Python 3.10.4
    Si ves un mensaje de error o dice que no se reconoce el comando, entonces necesitas instalar Python.

2. Instalar Python

Si no tienes Python instalado o necesitas actualizarlo, puedes seguir los pasos adecuados para tu sistema operativo:

2.1. En Windows

  1. Accede al sitio web oficial: https://www.python.org/downloads/windows/
  2. Haz clic en el botón de descarga de la última versión.
  3. Ejecuta el archivo descargado.
  4. En la ventana del instalador:
    • Marca la opción "Add Python to PATH".
    • Haz clic en "Install Now".
  5. Una vez instalada, puedes verificarlo abriendo la consola (cmd) y escribiendo: 
    python --version

2.2. En macOS

  1. Descarga el instalador desde: https://www.python.org/downloads/macos/
  2. Abre el archivo .pkg descargado y sigue los pasos del instalador.
  3. Para verificar la instalación, abre la Terminal y escribe: 
    python3 --version

2.3. En Linux

La mayoría de distribuciones Linux ya incluyen Python. Para verificarlo, abre la terminal y escribe:

python3 --version

Si no está instalado o deseas actualizarlo, puedes usar el gestor de paquetes correspondiente. Por ejemplo, en Debian/Ubuntu:

    sudo apt update
    sudo apt install python3

3. Instalar pip

pip es una herramienta que permite instalar paquetes y librerías adicionales en Python. Normalmente, se instala automáticamente junto con Python. Aquí aprenderás cómo verificar si ya está instalado y cómo instalarlo en caso de que no lo esté.

3.1 Verificar si pip está instalado

  1. Abre una terminal o consola de comandos.
  2. Escribe el siguiente comando y presiona Enter: 
    pip --version
    o en algunos sistemas: 
    pip3 --version
  3. Si ves un mensaje con la versión de pip (por ejemplo, pip 23.1.2), entonces ya está instalado.
  4. Si aparece un mensaje de error indicando que el comando no se reconoce, deberás instalar pip manualmente.

3.2 Instalar pip (si es necesario)

En Windows o macOS (si no se instaló con Python)

  1. Descarga el script get-pip.py desde el sitio oficial: https://bootstrap.pypa.io/get-pip.py
  2. Guarda el archivo en una ubicación fácil de acceder, por ejemplo en el escritorio.
  3. Abre una terminal o consola, navega a la carpeta donde está guardado el archivo y ejecuta: 
    python get-pip.py

En Linux (usando el gestor de paquetes)

En la mayoría de distribuciones de Linux puedes instalar pip con el gestor de paquetes. Por ejemplo, en Debian/Ubuntu:

sudo apt update
  sudo apt install python3-pip

Luego verifica la instalación con:

pip3 --version

Una vez instalado pip, podrás usarlo para instalar paquetes de Python, como por ejemplo:

pip install nombre_del_paquete

4. Configurar entorno virtual

Un entorno virtual en Python es una herramienta que permite crear un espacio aislado para instalar dependencias y paquetes específicos para un proyecto, sin afectar otras configuraciones del sistema.

Para evitar problemas con dependencias entre proyectos y tener un entorno aislado para cada uno de tus proyectos, es recomendable usar entornos virtuales. Además, en Linux, si no configuras un entorno virtual, tendrás problemas con pip . (si estás en Windows, te puedes saltar este paso)

Instalar la herramienta para crear entornos virtuales - VENV

$ sudo apt install python3-venv

4.1 Crear un entorno virtual

  1. Abre una terminal o consola de comandos.
  2. Ubícate en la carpeta donde quieres crear el entorno virtual o crea una nueva: 
    mkdir mi_proyecto
  cd mi_proyecto
  3. Ejecuta el siguiente comando para crear el entorno virtual (por convención lo llamaremos mientorno): 
    python -m venv mientorno

    Este comando creará una carpeta llamada mientorno que contendrá el entorno virtual.

4.2 Activar el entorno virtual

En Windows

mientorno\Scripts\activate

En macOS o Linux

source mientorno/bin/activate

Después de activar el entorno virtual, deberías ver el nombre del entorno (por ejemplo, (mientorno)) al inicio de la línea en la consola, indicando que está activo.

4.3 Verificar instalación de pip en el entorno

Una vez activado el entorno, asegúrate de que pip esté disponible:

pip --version

A partir de aquí, ya se pueden instalar paquetes dentro del entorno virtual

4.4 Desactivar el entorno virtual

Cuando termines de trabajar en tu proyecto, puedes salir del entorno virtual ejecutando:

deactivate

Esto te devolverá al entorno global del sistema.

5. Instalar paquetes

En Python, los paquetes son módulos reutilizables que extienden las funcionalidades del lenguaje. Para gestionar e instalar estos paquetes se utiliza pip, el gestor oficial de paquetes de Python.

5.1 Instalar un paquete con pip

Para instalar un paquete, utiliza el siguiente comando en la consola (preferentemente con el entorno virtual activado)

pip install nombre_del_paquete

Ejemplo:

pip install requests

Ejemplo:

pip install pyfiglet
      uso:   pyfiglet hola python

5.2 Instalar múltiples paquetes desde un archivo

También es posible instalar varios paquetes definidos en un archivo requirements.txt con el siguiente comando:

pip install -r requirements.txt

Este archivo contiene una lista de paquetes, uno por línea, que el proyecto necesita.

5.3 Ver paquetes instalados

Para ver qué paquetes están instalados en tu entorno actual:

pip list

5.4 Desinstalar un paquete

Si necesitas desinstalar un paquete:

pip uninstall nombre_del_paquete

5.5 Actualizar un paquete

Para actualizar un paquete a su última versión:

pip install --upgrade nombre_del_paquete

6. Instalar un editor de texto o un IDE

Para escribir y ejecutar código en Python de manera eficiente, es recomendable usar un editor de texto avanzado o un IDE (Entorno de Desarrollo Integrado). Estas herramientas ofrecen características como resaltado de sintaxis, autocompletado, depuración y gestión de proyectos.

6.1 Visual Studio Code (VS Code)

Visual Studio Code es uno de los editores más populares para desarrollo en Python. Es liviano, extensible y gratuito.

  1. Descárgalo desde: https://code.visualstudio.com/
  2. Instálalo como cualquier otra aplicación para tu sistema operativo.
  3. Una vez abierto, instala la extensión oficial de Python:
    • Haz clic en el icono de extensiones (barra lateral izquierda).
    • Busca “Python” (editor: Microsoft) e instálala.
  4. Con la extensión instalada, podrás ejecutar scripts, usar depuración, y trabajar con entornos virtuales desde VS Code.

6.2 Otros editores o IDEs populares

  • PyCharm: IDE profesional desarrollado por JetBrains. Disponible en versión gratuita (Community) y de pago. Web: https://www.jetbrains.com/pycharm/
  • Sublime Text: Editor de texto rápido y personalizable. Web: https://www.sublimetext.com/
  • Jupyter Notebook: Ideal para proyectos de ciencia de datos. Se instala con pip: 
    pip install notebook
    Luego se ejecuta con: 
    jupyter notebook

6.3 Recomendación

Para quienes están comenzando, Visual Studio Code es una excelente opción por su simplicidad y potencia. Además, permite integrar control de versiones con Git y múltiples herramientas para desarrollo moderno.

7. Configuración de Linters y Formateadores

Los linters y formateadores ayudan a mejorar la calidad del código. Los linters detectan errores, advertencias o malas prácticas, mientras que los formateadores corrigen automáticamente el estilo del código para que sea más legible y uniforme.

7.1 Herramientas recomendadas

  • Flake8: Linter que verifica errores de sintaxis y estilo.
  • Black: Formateador de código que aplica reglas de estilo consistentes.
  • isort: Organiza automáticamente las importaciones en el código.

7.2 Instalación de herramientas

Con tu entorno virtual activado, instala las herramientas con pip:

pip install flake8 black isort

7.3 Configuración en Visual Studio Code

  1. Abre el menú de configuración (⚙️ → "Settings").
  2. Busca Python Linting y habilita la opción Linting: Enabled.
  3. Selecciona flake8 como linter predeterminado.
  4. Para configurar el formateador, busca Python Formatting Provider y selecciona black.
  5. Habilita el formato al guardar (opcional pero recomendado):
    • Busca "Format On Save" y actívalo.

7.4 Uso manual desde la terminal

También puedes ejecutar estas herramientas desde la consola:

  • flake8 para analizar el código: 
    flake8 nombre_archivo.py
  • black para formatear automáticamente: 
    black nombre_archivo.py
  • isort para ordenar importaciones: 
    isort nombre_archivo.py

7.5 Archivos de configuración (opcional)

Puedes crear archivos como pyproject.toml o .flake8 para definir configuraciones personalizadas para cada herramienta.

Ejemplo de pyproject.toml para Black:

  [tool.black]
  line-length = 88
  target-version = ['py39']

8. Depuración

Para depurar tu código Python, puedes usar herramientas como pdb, que es el depurador integrado en Python. Solo necesitas incluir la línea: import pdb; pdb.set_trace() en tu código para comenzar a depurar.

9. Control de versiones

El control de versiones permite gestionar los cambios en el código fuente de manera eficiente. La herramienta más utilizada para esto es Git, que permite mantener un historial de versiones, colaborar con otras personas y trabajar en múltiples ramas del proyecto.

9.1 Instalar Git

Descarga Git desde su sitio oficial:

https://git-scm.com/

La instalación es sencilla y depende del sistema operativo. En Windows, incluye una terminal llamada Git Bash.

9.2 Configuración inicial

Después de instalar Git, es recomendable configurar tu nombre de usuario y correo electrónico:

  git config --global user.name "Tu Nombre"
  git config --global user.email "tu.email@example.com"

9.3 Comandos básicos de Git

  • Inicializar un repositorio: 
    git init
  • Ver el estado del repositorio: 
    git status
  • Agregar archivos al área de preparación: 
    git add nombre_archivo.py
  git add .    -- Agrega todos los archivos modificados --
  • Hacer un commit (guardar los cambios): 
    git commit -m "Mensaje descriptivo del cambio"
  • Ver el historial de commits: 
    git log

9.4 Usar GitHub (repositorio remoto)

GitHub es una plataforma para alojar repositorios Git en la nube. Permite colaborar y respaldar proyectos.

  1. Crea una cuenta en https://github.com/
  2. Crea un nuevo repositorio desde la web.
  3. Conecta tu repositorio local con GitHub: 
    
  git remote add origin https://github.com/usuario/nombre-repo.git
  git push -u origin master

9.5 .gitignore

El archivo .gitignore indica a Git qué archivos o carpetas no deben incluirse en el control de versiones (por ejemplo, entornos virtuales, archivos temporales, etc.).

Ejemplo de contenido de .gitignore:

  venv/
  __pycache__/
  *.pyc
  .env

Con estas herramientas y buenas prácticas, podrás mantener un proyecto bien organizado y controlado.

10. Configuración de entornos seguros

Cuando se trabaja en proyectos de software, es importante mantener un entorno seguro para proteger credenciales, datos sensibles y evitar errores por mal manejo de configuraciones. A continuación se presentan buenas prácticas para lograrlo.

10.1 Variables de entorno

Las variables de entorno permiten almacenar información sensible, como claves API o contraseñas, sin incluirlas directamente en el código fuente.

Para gestionar estas variables, se puede usar un archivo .env junto con la librería python-dotenv:

pip install python-dotenv

Ejemplo de un archivo .env:

  SECRET_KEY=mi_clave_secreta
  DEBUG=False

Y para cargarlo en tu aplicación Python:

  from dotenv import load_dotenv
  import os
  load_dotenv()
  clave = os.getenv("SECRET_KEY")

10.2 Uso del archivo .gitignore

Es importante asegurarse de que archivos como .env, archivos de configuración local y entornos virtuales no se suban a repositorios públicos:

  .env
  venv/
  __pycache__/
  *.pyc

10.3 Gestión de dependencias seguras

  • Utiliza entornos virtuales para aislar tus proyectos.
  • Congela las versiones de tus paquetes con: 
    pip freeze > requirements.txt
  • Revisa actualizaciones y vulnerabilidades en dependencias con herramientas como pip-audit: 
    pip install pip-audit
  pip-audit

10.4 Control de acceso y buenas prácticas

  • No compartas credenciales por correo ni en repositorios.
  • Usa autenticación en dos pasos en plataformas como GitHub.
  • Aplica principios de mínimo privilegio en tus entornos.

Estas prácticas contribuyen a desarrollar aplicaciones más seguras y profesionales, especialmente en entornos colaborativos o de producción.
